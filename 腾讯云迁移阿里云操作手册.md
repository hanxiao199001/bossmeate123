# BossMate 腾讯云 → 阿里云 迁移操作手册

> 腾讯云：106.53.163.120 | 阿里云：47.82.159.93
> 迁移方式：阿里云直接从腾讯云拉取（服务器直传）
> 项目目录：/home/projects/bossmate/

---

## 第一步：在腾讯云上确认和打包

SSH 连接腾讯云：
```bash
ssh root@106.53.163.120
```

### 1.1 确认项目目录结构
```bash
ls -la /home/projects/bossmate/
```

### 1.2 查看当前 Docker 容器状态（记录下来以便参考）
```bash
docker ps -a
docker-compose -f /home/projects/bossmate/docker-compose.yml config
```

### 1.3 导出 PostgreSQL 数据库
```bash
# 先看数据库容器名
docker ps | grep postgres

# 导出数据库（把 CONTAINER_NAME 替换成实际容器名）
docker exec CONTAINER_NAME pg_dump -U bossmate -d bossmate > /home/projects/bossmate/backup_db.sql

# 确认导出成功
ls -lh /home/projects/bossmate/backup_db.sql
```

### 1.4 记录环境变量
```bash
cat /home/projects/bossmate/.env
```
> ⚠️ 把这个内容复制保存下来，阿里云要用

### 1.5 确认上传文件和 LanceDB 数据
```bash
ls -lh /home/projects/bossmate/data/
du -sh /home/projects/bossmate/data/*
```

## 第二步：在阿里云上拉取项目

另开一个终端窗口，SSH 连接阿里云：
```bash
ssh root@47.82.159.93
```

### 2.1 创建项目目录
```bash
mkdir -p /home/projects
```

### 2.2 从腾讯云直接拷贝整个项目（含数据库备份）
```bash
scp -r root@106.53.163.120:/home/projects/bossmate /home/projects/
```
> 会提示输入腾讯云 root 密码，输入后等待传输完成
> 如果文件很大，耐心等一下

### 2.3 确认文件传输完成
```bash
ls -la /home/projects/bossmate/
du -sh /home/projects/bossmate/
```

## 第三步：阿里云环境配置

### 3.1 确认 Docker 和 Docker Compose 已安装
```bash
docker --version
docker compose version
```
> 如果没安装，运行：
> ```bash
> curl -fsSL https://get.docker.com | sh
> systemctl enable docker && systemctl start docker
> ```

### 3.2 修改环境变量（如果需要调整）
```bash
nano /home/projects/bossmate/.env
```
> 一般不需要改，数据库连接走的是 Docker 内部网络

### 3.3 清理不需要的文件
```bash
# 删除 node_modules（会在 Docker 构建时重新安装）
rm -rf /home/projects/bossmate/node_modules
rm -rf /home/projects/bossmate/packages/*/node_modules
rm -rf /home/projects/bossmate/apps/*/node_modules
```

---

## 第四步：启动 Docker 服务

### 4.1 进入项目目录
```bash
cd /home/projects/bossmate
```

### 4.2 启动基础服务（PG + Redis）
```bash
docker compose up -d postgres redis
```

### 4.3 等待数据库就绪（约10秒），然后导入数据
```bash
# 等数据库启动
sleep 10

# 确认 postgres 容器在跑
docker compose ps

# 导入数据库备份（把 CONTAINER_NAME 替换成实际容器名）
cat /home/projects/bossmate/backup_db.sql | docker exec -i CONTAINER_NAME psql -U bossmate -d bossmate
```

### 4.4 启动全部服务
```bash
docker compose up -d
```

### 4.5 查看所有容器状态
```bash
docker compose ps
```

### 4.6 查看应用日志，确认没有报错
```bash
docker compose logs --tail=50 app
```

## 常见问题

**Q: scp 传输很慢怎么办？**
可以先在腾讯云打包压缩再传：
```bash
# 腾讯云上
cd /home/projects && tar czf bossmate.tar.gz bossmate/
# 阿里云上
scp root@106.53.163.120:/home/projects/bossmate.tar.gz /home/projects/
tar xzf /home/projects/bossmate.tar.gz -C /home/projects/
```

**Q: Docker Compose 文件需要修改吗？**
一般不需要。如果腾讯云用了特殊的网络配置或端口映射，可能需要微调。

**Q: 忘记了腾讯云的数据库密码？**
看 `.env` 文件或 `docker-compose.yml` 里的环境变量。

**Q: 阿里云安全组怎么配？**
阿里云控制台 → ECS → 安全组 → 入方向添加规则 → 开放 80/443/22 端口。
